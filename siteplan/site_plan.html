Player One Reviews Site Plan Document
01

Purpose & Audience

Purpose

Player One Reviews is a personal gaming review site where I document my honest, first-hand experiences with every game I play — across all genres and platforms. The site serves as a living archive of my gaming journey, offering well-considered ratings, spoiler-tagged analysis, and quick-reference verdict cards that help readers decide whether a game is worth their time.

The secondary goal is to build a searchable, filterable game log that I can look back on myself and share with friends.

Audience

The primary audience is fellow gamers — casual and hardcore — who value authentic, personal reviews over aggregated scores. Secondary visitors include friends and classmates who want game recommendations and anyone who lands via a search for a specific title. Readers are comfortable with gaming terminology and expect snappy prose, not academic writing.

02

Dynamic Elements JavaScript

  • Live Search & Filter A real-time search bar filters the review card grid by title, genre, or platform as the user types — no page reload needed.
  • Star Rating Widget Hoverable, clickable star ratings on each review card animate in sequence and display a numeric score tooltip on hover.
  • Spoiler Toggle Story-sensitive review sections are hidden by default. A JS-powered button reveals them with a smooth slide-down animation when clicked.
  • Dark / Light Mode Toggle A theme toggle button switches the site between a clean white mode and a dark slate mode, saving the user's preference to localStorage.
  • Back-to-Top Button A floating button appears via a scroll-event listener once the user scrolls past 300 px, and smoothly returns them to the top.
  • Animated Score Counter On page load, the overall score numbers count up from 0 to their final value, creating a satisfying reveal effect.

Logo

The logo combines a compact game-controller icon on a solid dark tile with the wordmark set in Syne 800. The controller icon uses a minimal two-color palette (ink black + accent red) that echoes the site's button-accent color. The icon works at all sizes — from browser tab favicon down to 16 px.

Sample Review Card Data

Each review card will include the following data fields:

  • title — Game title (string)
  • platform — e.g. PC, PS5, Switch (string)
  • genre — e.g. RPG, FPS, Strategy (string)
  • score — Numeric 1–10 (number)
  • verdict — One-line summary (string)
  • coverImage — Path to screenshot or key art (string)
  • dateReviewed — ISO date string
  • hoursPlayed — Approximate hours (number)
  • pros — Array of bullet strings
  • cons — Array of bullet strings

Images & Assets

  • Game cover art — sourced from official press kits or IGDB API
  • Screenshots — captured personally during gameplay
  • Icons (star, controller, platform logos) — inline SVG or open-license icon set
  • Author photo — personal photograph
